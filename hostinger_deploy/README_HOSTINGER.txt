خطوات تشغيل النظام على استضافة Hostinger وحل مشكلة 403:

1. الرفع (Upload):
   - قم برفع كافة محتويات هذا المجلد (وليس المجلد نفسه) إلى داخل مجلد الموقع في الاستضافة (عادة public_html).

2. إعداد Node.js في لوحة تحكم Hostinger:
   - ابحث عن "Node.js" في لوحة تحكم Hostinger (hPanel).
   - قم بإنشاء تطبيق جديد أو تعديل الحالي.
   - اختر نسخة Node.js: يفضل نسخة 20 أو أحدث.
   - ملف البداية (Application startup file): يجب أن يكون server.js.
   - رابط التطبيق (Application URL): تأكد أنه يشير إلى accounts.iw-om.com.

3. تثبيت المكتبات (Install Dependencies):
   - بعد رفع الملفات، اضغط على زر "npm install" في لوحة تحكم Node.js في Hostinger.
   - أو استخدم الـ Terminal ونفذ: npm install

4. بناء المشروع (Build):
   - في لوحة تحكم Node.js أو الـ Terminal، نفذ الأمر التالي: npm run build
   - هذا الأمر سيقوم بإنشاء مجلد .next الضروري للتشغيل.

5. التشغيل (Start):
   - اضغط على زر "Start" أو "Restart" لتشغيل التطبيق.

6. حل مشكلة 403 Forbidden:
   - تأكد أن ملف .htaccess موجود في المجلد الرئيسي (public_html).
   - تأكد أن التطبيق "Started" في لوحة التحكم. إذا كان متوقفاً، سيعطيك السيرفر خطأ 403 لأن Apache لا يجد ملف index.
   - تأكد من صلاحيات المجلدات (755) والملفات (644).

ملاحظة هامة:
- تم ضبط قاعدة البيانات لتعمل على (localhost) لتكون أسرع وأكثر أماناً داخل الاستضافة.
- الملف server.js مصمم ليعمل تلقائياً مع نظام standalone الخاص بـ Next.js لضمان أفضل أداء.
